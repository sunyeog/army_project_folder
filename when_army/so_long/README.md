# so_long

학교 그래픽 라이브러리인 MiniLibX로 만든 간단한 2D 게임입니다.

## Description

so_long은 타일 기반 맵 위에서 플레이어를 움직여, 맵의 모든 수집품을 모은 뒤
출구로 빠져나가는 탑다운 2D 게임입니다. 맵은 `.ber` 파일에서 읽어들여 검증한
뒤, MiniLibX 이미지로 창에 렌더링됩니다.

이 프로젝트의 목표는 C에서 창 관리, 이벤트 처리, 텍스처, 그리고 기본적인 맵
검증 알고리즘을 다뤄보는 것입니다.

**게임 규칙**
- 맵의 모든 수집품(`C`)을 모은 뒤 출구(`E`)에 도달하면 게임이 종료됩니다.
- 수집품이 남아 있는 동안에도 출구 칸 위를 지나갈 수는 있지만, 게임은
  종료되지 않습니다.
- 벽(`1`)으로는 이동할 수 없습니다.
- 실제로 이동에 성공한 경우에만 이동 횟수가 셸에 출력됩니다.

## 맵 형식

맵은 다섯 가지 문자로 이루어진 `.ber` 텍스트 파일입니다.

| 문자 | 의미          |
|------|---------------|
| `0`  | 빈 공간        |
| `1`  | 벽            |
| `C`  | 수집품         |
| `E`  | 출구          |
| `P`  | 플레이어 시작   |

유효한 맵의 조건:
- 위 다섯 문자 외의 문자가 없어야 합니다.
- 빈 줄이 없어야 합니다.
- 직사각형이어야 합니다.
- 사방이 벽으로 둘러싸여 있어야 합니다.
- `P`가 정확히 하나, `E`가 정확히 하나, `C`가 하나 이상 있어야 합니다.
- 플레이어가 모든 수집품과 출구에 도달할 수 있는 경로가 존재해야 합니다.

조건을 만족하지 않으면 `Error\n`와 함께 원인을 설명하는 메시지를 출력하고
정상 종료합니다.

예시:
```
1111111111111
10010000000C1
1000011111001
1P0011E000001
1111111111111
```

## 조작

| 키              | 동작        |
|-----------------|-------------|
| `W` / `↑`       | 위로 이동    |
| `S` / `↓`       | 아래로 이동  |
| `A` / `←`       | 왼쪽으로 이동 |
| `D` / `→`       | 오른쪽으로 이동 |
| `ESC`           | 종료        |
| 창 닫기 버튼     | 종료        |

## 실행 방법

**빌드**
```
make
```
MiniLibX와 libft를 각자의 Makefile로 컴파일한 뒤 `so_long` 실행 파일을
생성합니다. 사용 가능한 규칙: `all`, `clean`, `fclean`, `re`.

**실행**
```
./so_long map.ber
```
유효한 `.ber` 맵 파일 하나를 인자로 넘깁니다.

| 파일                    | 역할                                   |
|-------------------------|----------------------------------------|
| `so_long.c`             | 진입점, 시작 위치·크기 탐색, 배열 해제  |

## 기술 노트

**경로 검증**
맵을 복제한 사본 위에서 플러드 필(DFS)을 수행합니다. 플레이어 위치에서
시작해 벽이 아닌 칸을 모두 방문 처리한 뒤, 방문되지 않은 `C`나 `E`가 남아
있으면 도달 불가능한 맵으로 판단해 에러를 냅니다. 원본 맵은 그대로 두어야
하므로 사본을 만들어 사용합니다.

**출구 위를 지나가기**
플레이어는 수집품을 다 모으기 전에도 출구 칸 위를 지나갈 수 있습니다.
이를 위해 `t_game`에 `under` 필드를 두어, 플레이어가 서 있는 칸의 원래 값을
기억합니다. 플레이어가 그 칸을 떠날 때 `under`에 저장된 값으로 복원하므로
출구가 사라지지 않습니다. 수집품을 밟은 경우에는 `under`를 `'0'`으로 두어
수집품이 사라지도록 했습니다.

**이미지 관리**
텍스처는 매 프레임 다시 불러오지 않고 시작 시 한 번만 로드해 재사용합니다.
렌더링마다 이미지를 새로 로드하면 그만큼 메모리가 계속 쌓이기 때문입니다.
종료 시에는 이미지·창·디스플레이·인스턴스를 모두 해제합니다.

**종료 처리**
ESC, 게임 클리어, 창 닫기 세 경로 모두 `end_game()` 하나로 모았습니다.
이 함수가 모든 자원을 해제하고 직접 `exit()`을 호출하므로, 해제된 맵에
다시 접근하는 일이 없습니다.

**메모리**
정상 종료와 에러 경로 양쪽 모두에서 힙 할당을 해제합니다. 에러 처리 함수는
`t_game` 포인터를 받아 맵이 이미 할당된 경우에만 해제하며, 호출 지점에서
이미 직접 해제한 경우에는 `NULL`을 넘겨 이중 해제를 피합니다.
